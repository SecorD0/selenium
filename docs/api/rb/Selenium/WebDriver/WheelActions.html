#scroll(x, y, origin: ScrollOrigin.viewport(0, 0), device: nil) ⇒ ActionBuilder

Scrolls the viewport based on a ScrollOrigin.

This method is needed instead of #scroll_to or #scroll_by when what needs to be scrolled is only in a portion of the viewport. The origin can be thought of as where on the screen you put the mouse when executing a wheel scroll, or where you put your cursor when swiping a touch pad, etc.

The offset for the origin is referenced to either the upper left of the viewport or the center of the element The methods ScrollOrigin.viewport and ScrollOrigin.element are provided to ensure correct syntax

Examples:

Scroll by the provided amount originating from a source offset from upper left of the viewport


el = driver.find_element(id: "some_id")
driver.action.scroll(0, 100, ScrollOrigin.viewport(400, 200)).perform

Scroll by the provided amount originating from a source offset from the center of the provided element


el = driver.find_element(id: "some_id")
driver.action.scroll(0, 100, ScrollOrigin.element(element, x: -400, 100)).perform

Parameters:

  • x (Integer) —

    horizontal offset. A negative value means scrolling left.

  • y (Integer) —

    vertical offset. A negative value means scrolling up.

  • origin (Hash) (defaults to: ScrollOrigin.viewport(0, 0)) —

    The location the scroll originates from

  • device (Symbol || String) (defaults to: nil) —

    Optional name of the WheelInput device to scroll with

Returns:

  • (ActionBuilder) —

    A self reference.

Raises:

  • (MoveTargetOutOfBoundsError) —

    if the origin value is outside the viewport.

See Also:

  • ScrollOrigin

def scroll(x, y, origin: ScrollOrigin.viewport(0, 0), device: nil)
  wheel = wheel_input(device)
  opts = {delta_x: Integer(x),
          delta_y: Integer(y),
          duration: default_scroll_duration}.merge!(origin)
  wheel.create_scroll(**opts)
  tick(wheel)
  self
end

#scroll_to(element, x = 0, y = 0, device: nil) ⇒ ActionBuilder

Scrolls the viewport so that the provided element is at the bottom. Then the viewport is further scrolled by the provided x and y offsets.

Examples:

Scroll to element


el = driver.find_element(id: "some_id")
driver.action.scroll_to(el).perform

Scroll to offset from element


el = driver.find_element(id: "some_id")
driver.action.scroll_to(el, 0, 1000).perform

Parameters:

  • element (Selenium::WebDriver::Element) —

    to scroll to.

  • x (Integer) (defaults to: 0) —

    Optional horizontal offset to scroll from the center of the element. A negative value means scrolling left.

  • y (Integer) (defaults to: 0) —

    Optional vertical offset to scroll from the center of the element. A negative value means scrolling up.

  • device (Symbol || String) (defaults to: nil) —

    optional name of the WheelInput device to scroll with.

Returns:

  • (ActionBuilder) —

    A self reference.
