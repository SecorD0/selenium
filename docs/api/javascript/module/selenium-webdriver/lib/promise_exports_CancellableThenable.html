Marker interface for objects that allow consumers to request the cancellation of a promise-based operation. A cancelled promise will be rejected with a CancellationError.

This interface is considered package-private and should not be used outside of selenium-webdriver.

this.cancel( opt_reason ) → void

Requests the cancellation of the computation of this promise's value, rejecting the promise in the process. This method is a no-op if the promise has already been resolved.

Parameters
opt_reason(string|Error|null|undefined)

The reason this promise is being cancelled. This value will be wrapped in a CancellationError.

this.catch<R>( errback ) → Thenable<(R|null)>

Registers a listener for when this promise is rejected. This is synonymous with the catch clause in a synchronous API:

 // Synchronous API:
 try {
   doSynchronousWork();
 } catch (ex) {
   console.error(ex);
 }

 // Asynchronous promise API:
 doAsynchronousWork().catch(function(ex) {
   console.error(ex);
 });
Parameters
errbackfunction(*): (R|IThenable<(R|null)>|null)

The function to call if this promise is rejected. The function should expect a single argument: the rejection reason.

Returns
Thenable<(R|null)>

A new promise which will be resolved with the result of the invoked callback.
